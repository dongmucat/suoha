# Product Brief: suoha

## Executive Summary

suoha（梭哈）是一个为线下德州扑克玩家打造的免费 Web 应用，解决没有实体筹码时记账结算的痛点。它采用极简设计哲学，通过房间机制实现多人实时筹码同步，让玩家专注于游戏本身而非繁琐的账目管理。

核心价值：**轻松、放心、简单好用**——打开浏览器即用，不存历史数据，基于朋友间互信的全透明记账模式。

---

## Core Vision

### Problem Statement

线下德州扑克玩家经常遇到没有实体筹码的情况，导致游戏体验很别扭。虽然可以用纸笔、计算器或其他替代方式记账，但这些方法要么容易出错、要么操作繁琐、要么不够透明，所有参与者都感到不便。

市面上虽然存在类似的记账工具，但大多需要付费，且功能复杂，不符合"随时可用、简单直接"的需求。

### Problem Impact

**对玩家的影响：**
- 游戏体验受损：无法专注于牌局本身，需要分心处理账目
- 信任问题：手工记账容易产生争议和误解
- 门槛提高：没有筹码就无法顺利开局，降低了游戏的便利性

**发生频率：**
虽然不是高频场景，但对于经常玩德州扑克的固定牌友圈来说，这是一个真实存在且影响体验的痛点。

### Why Existing Solutions Fall Short

**现有方案的局限性：**

1. **收费产品**：市面上的类似工具需要付费，增加了使用门槛
2. **功能过载**：很多产品功能复杂，学习成本高，不够"即用即走"
3. **需要下载**：需要安装 app，不如 Web 应用随时可用
4. **过度设计**：存储历史、复杂权限、繁琐流程，违背了"够用就好"的原则

### Proposed Solution

suoha 是一个**免费、极简、基于 Web 的德州扑克筹码记账系统**，核心特性包括：

**核心功能：**
- **房间机制**：房主创建房间生成二维码和房间号，其他玩家扫码或输入房间号加入（最多8人）
- **自助筹码操作**：每个玩家独立操作自己的筹码（下注、收回、平分池底），无需审批
- **实时同步**：通过 WebSocket 实现所有玩家的筹码变动和池底状态实时可见
- **一键结算**：房主触发结算，显示每人盈亏清单和总账校验

**设计哲学：**
- **极简主义**：不存历史、不搞复杂权限、不管牌局逻辑，只做记账和结算
- **信任优先**：基于朋友间互信，全透明操作，总账校验保证公平
- **即用即走**：Web 应用，打开浏览器就能用，用完即走，下次重新开始

**技术实现：**
- 前端：Vite + React（H5）
- 后端：Java
- 实时通信：WebSocket

### Key Differentiators

1. **完全免费**：无需付费，随时可用
2. **Web 化**：不需要下载 app，手机浏览器打开即用
3. **极致简化**：
   - 允许负数筹码，无需"买入"概念
   - 不存储历史数据，每局独立
   - 自助操作，无需复杂权限管理
4. **信任模型**：全透明 + 总账校验，适合熟人局
5. **即用即走**：30秒开局，用完即走，下次继续

**核心价值主张：**
让线下德州扑克玩家**不再为没有筹码发愁**，专注于游戏本身，享受轻松、放心、简单的记账体验。

## Target Users

### Primary Users

**用户画像：休闲德州扑克玩家**

**背景与特征：**
- 同学或朋友关系，偶尔参与周末聚会
- 对德州扑克不太熟练，新手居多
- 使用频率：约一个月一次
- 使用场景：清吧或家里的朋友聚会
- 技术接受度：习惯使用手机浏览器，但希望工具简单直接

**问题体验：**
- 临时起意想玩德州扑克，但没有实体筹码
- 不熟悉规则，需要快速参考基本玩法
- 手工记账容易出错，结算时容易产生争议
- 希望专注于游戏本身，而不是繁琐的账目管理

**成功愿景：**
- 打开浏览器就能用，30秒内开始游戏
- 操作简单直观，新手也能快速上手
- 实时看到筹码变动，透明公平
- 结算清晰准确，不用担心算错账
- 用完即走，下次继续用

**典型用户：小李（普通玩家）**
- 25-30岁，偶尔参加朋友聚会
- 玩过几次德州扑克，但不是很熟练
- 希望工具简单好用，不想学习复杂功能
- 关键需求：快速加入房间、自助操作筹码、查看规则提示

### Secondary Users

**房主用户**

**角色定位：**
- 通常是固定的组局者（如产品创建者本人）
- 负责创建房间、管理牌局生命周期、触发结算

**额外职责：**
- 创建房间并分享二维码或房间号
- 控制牌局的开始和结束
- 触发结算并确保账目清晰
- 必要时转让房主权限

**成功愿景：**
- 快速创建房间（30秒内完成）
- 方便管理玩家和牌局状态
- 一键结算，清晰的盈亏报告
- 总账校验确保公平性

### User Journey

**典型使用流程：**

1. **开局前（Discovery）**
   - 场景：周末朋友聚会，有人提议玩德州扑克，但没有实体筹码
   - 触发：房主提出"用 suoha 来记账吧"

2. **创建与加入（Onboarding）**
   - 房主打开 suoha，30秒创建房间，生成二维码和房间号
   - 其他玩家扫码或输入房间号加入（最多8人）
   - 新手点击"规则提示"按钮，快速了解德州扑克基本玩法

3. **游戏中（Core Usage）**
   - 每个玩家自己操作筹码：下注（快捷按钮或自定义金额）、收回池底、平分池底
   - 实时同步：所有人看到筹码变动和池底状态
   - 全透明：每个人的筹码数和下注明细实时可见

4. **成功时刻（Aha Moment）**
   - 第一次顺利完成一局游戏，不用担心算错账
   - 结算时看到清晰的盈亏清单和总账校验，大家都认可结果
   - 意识到"终于不用再为没有筹码发愁了"

5. **长期使用（Long-term）**
   - 下次聚会时，自然而然地再次使用 suoha
   - 成为固定牌友圈的标配工具
   - 用完即走，不需要维护历史数据

**关键交互点：**
- 房主：创建房间 → 管理牌局 → 触发结算
- 普通玩家：加入房间 → 查看规则（可选）→ 操作筹码 → 查看结算

## Success Metrics

### 用户成功指标

**核心体验目标：流畅、无 bug**

1. **快速开局**
   - 创建房间时间 ≤ 30 秒（从打开页面到生成二维码）
   - 加入房间时间 ≤ 10 秒（扫码或输入房间号到进入房间）
   - 目标：让玩家快速开始游戏，不被工具拖累

2. **实时同步**
   - 筹码操作同步延迟 ≤ 1 秒
   - 所有玩家看到的数据实时一致
   - 目标：确保游戏体验流畅，避免混乱

3. **首次使用成功率**
   - 新用户首次使用完整流程成功率 = 100%
   - 完整流程：创建/加入房间 → 操作筹码 → 查看结算
   - 目标：新手也能顺利使用，无需学习成本

4. **数据准确性**
   - 总账校验通过率 = 100%（盈亏之和必须为 0）
   - 筹码计算无误差
   - 目标：确保公平性，避免争议

5. **连接稳定性**
   - WebSocket 连接稳定，游戏过程中不掉线
   - 如掉线，能自动重连并恢复状态
   - 目标：不因技术问题中断游戏

6. **操作简单性**
   - 新手在 5 分钟内能学会基本操作
   - 核心操作（下注、收回、平分）一目了然
   - 目标：降低学习成本，提升易用性

**个人工具目标：满足自用需求，长期可维护**

**3 个月目标：**
- 核心功能稳定可用，无严重 bug
- 成功支持至少 3 次真实牌局
- 工具成为固定牌友圈的标配

**12 个月目标：**
- 工具持续稳定运行，维护成本低
- 支持更多朋友圈使用（如有需求）
- 可能的功能优化（基于实际使用反馈）

**长期愿景：**
- 成为线下德州扑克的便捷记账工具
- 保持极简设计，不过度复杂化
- 随时可用，无需维护历史数据

### Key Performance Indicators

**核心 KPI（按优先级排序）：**

1. **首次使用成功率 = 100%**
   - 衡量方法：新用户完成完整流程（创建/加入 → 操作 → 结算）的成功率
   - 目标：确保新手也能顺利使用

2. **实时同步延迟 ≤ 1 秒**
   - 衡量方法：从操作发起到其他玩家看到变化的时间
   - 目标：确保流畅的游戏体验

3. **总账校验通过率 = 100%**
   - 衡量方法：每次结算时，盈亏之和是否为 0
   - 目标：确保数据准确性和公平性

4. **房间创建时间 ≤ 30 秒**
   - 衡量方法：从打开页面到生成二维码的时间
   - 目标：快速开局，不拖累游戏节奏

5. **WebSocket 连接稳定性 ≥ 99%**
   - 衡量方法：游戏过程中连接保持时间占比
   - 目标：避免因技术问题中断游戏

**次要 KPI：**

- 重复使用率：下次聚会时再次使用的概率
- 推荐率：用户主动推荐给其他朋友的概率
- 操作效率：平均每次筹码操作的步骤数和时间

## MVP Scope

### MVP 核心功能

**模块 1：用户系统**
- 手机号 + 密码登录，不存在则自动注册
- 一个界面完成登录/注册
- 一个用户同时只能在一个房间里

**模块 2：房间系统**
- 房主创建房间，生成纯数字房间号 + 二维码
- 扫码或输入房间号加入，最多 8 人
- 满员拒绝，提示报错
- 房主可转让权限给其他玩家
- 房主点"结束牌局"触发结算
- 所有人离开后房间自动销毁

**模块 3：筹码操作**
- 所有玩家初始筹码 0，可为负数
- 下注：快捷按钮 20/50/100/1000 + 自定义输入，需确认
- 收回：一键收回全部池底
- 平分池底：选人后按人数均分

**模块 4：池底系统**
- 系统维护公共池底
- 实时显示池底总额 + 每人下注明细

**模块 5：实时同步**
- WebSocket 推送，所有人实时看到筹码变动和池底变化
- 掉线后重新连接，能恢复之前的状态

**模块 6：结算**
- 房主触发，显示每人盈亏（正数赢/负数输）
- 总账校验（盈亏之和 = 0）
- 结算页关闭后不保留数据

**模块 7：辅助功能**
- 规则提示按钮：新手可查看德州扑克基本规则

### 技术栈

- 前端：Vite + React（H5）
- 后端：Java
- 实时通信：WebSocket
- 临时持久化：Redis（支持掉线恢复和房间状态）
- 部署：自有服务器

### 明确不做的（Out of Scope）

- 历史记录和数据持久化（结算后不保留）
- 复杂权限管理（只有房主和普通玩家两种角色）
- 牌局逻辑（不管发牌、比大小等）
- 社交功能（聊天、好友列表等）
- 微信登录、短信验证码等其他登录方式
- 多房间同时参与
- 其他桌游支持
- 商业化功能（广告、付费等）

### MVP 成功标准

- 能够支持 2-8 人完成一局完整的德州扑克记账和结算
- 创建房间 ≤ 30 秒，实时同步延迟 ≤ 1 秒
- 首次使用成功率 = 100%，总账校验通过率 = 100%
- 掉线后能自动重连并恢复状态

### Future Vision

**Phase 2（基于实际使用反馈）：**
- 功能优化：基于真实牌局中发现的体验问题
- 性能优化：基于实际使用中的性能瓶颈

**长期方向：**
- 可能扩展为通用多人实时记账工具，适用于其他桌游
- 保持极简设计哲学，不过度复杂化
