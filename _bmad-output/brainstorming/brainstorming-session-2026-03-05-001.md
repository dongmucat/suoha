## 阶段 2：模式识别（Mind Mapping）

### 模块划分

**模块 A：用户系统**
- 手机号 + 密码登录，不存在则自动注册

**模块 B：房间系统**
- 房主创建房间，生成纯数字房间号 + 二维码
- 扫码或输入房间号加入，最多 8 人
- 满员拒绝，提示报错
- 房主可转让权限给其他玩家
- 房主点"结束牌局"触发结算

**模块 C：筹码操作**
- 所有玩家初始筹码 0，可为负数
- 下注：快捷按钮 20/50/100/1000 + 自定义输入，需确认
- 收回：一键收回全部池底
- 平分池底：选人后按人数均分

**模块 D：池底系统**
- 系统维护公共池底
- 实时显示池底总额 + 每人下注明细

**模块 E：实时同步**
- WebSocket 推送，所有人实时看到筹码变动和池底变化

**模块 F：结算**
- 房主触发，显示每人盈亏（正数赢/负数输）
- 总账校验（盈亏之和 = 0）
- 结算页关闭后不保留数据

### 发现的模式

1. 极简哲学贯穿始终——每个决策都在做减法
2. 信任模型——朋友间互信，全透明 + 总账校验
3. 两个核心操作——下注和收回，系统不介入牌局过程
4. 房主是唯一管理角色——只多"结束牌局"和"转让房主"两个能力

## 阶段 3：想法深化（SCAMPER）

### SCAMPER 检验结果

- **S（替代）**：下注支持快捷金额 + 自定义输入
- **C（组合）**：收回默认收回全部池底，一键操作
- **A（调整）**：池底区域显示每人下注明细
- **M（修改）**：快捷金额先固定 20/50/100/1000
- **P（挪用）**：系统本质是多人实时记账工具，可适用于其他桌游
- **E（消除）**：去掉买入操作，初始 0 允许负数，极致简化
- **R（逆转）**：结算页只显示每人盈亏和总账校验，极简
